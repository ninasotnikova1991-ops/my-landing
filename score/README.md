# Своя Подача — PWA табло для пиклбола

Установочное веб-приложение (PWA) для клуба «Своя Кухня»: счёт, подача и
side-out по официальным правилам пиклбола. Работает офлайн, ставится иконкой на
домашний экран телефона.

## Что внутри

```
picklepoint/
├── index.html            ← само приложение (RU/EN, звуки, правило 0-0-2)
├── manifest.json         ← метаданные для установки (иконка, имя, цвета)
├── service-worker.js     ← офлайн-режим и кэш
├── icons/
│   ├── icon-192.png
│   ├── icon-512.png
│   ├── icon-maskable-192.png
│   ├── icon-maskable-512.png
│   ├── apple-touch-icon.png
│   └── favicon-64.png
└── README.md             ← этот файл
```

Все пути **относительные** — папку можно положить в подпапку сайта
(`.../score/`) или на отдельный домен, ничего в файлах не меняя.

## Публикация (твой случай: GitHub + Cloudflare Pages)

Сайт «Своя Кухня» уже в репозитории на GitHub и деплоится через Cloudflare
Pages, поэтому всё сводится к «скопировать папку и запушить».

1. Скопируй эту папку в корень репозитория сайта и **переименуй в `score`**
   (чтобы адрес был короткий: `.../score/`).
   Внутри репозитория должно получиться:
   ```
   <репозиторий-сайта>/
   ├── (файлы основного сайта)
   └── score/
       ├── index.html
       ├── manifest.json
       ├── service-worker.js
       └── icons/...
   ```
2. Закоммить и запушь:
   ```bash
   git add score
   git commit -m "Add Svoya Podacha scoreboard PWA"
   git push
   ```
3. Cloudflare Pages сам пересоберёт и задеплоит. Через ~1 минуту приложение
   будет по адресу:
   ```
   https://<твой-домен>/score/
   ```

Готово. Никаких настроек сборки менять не нужно — это статические файлы, а пути
относительные.

> Почему подпапка, а не корень сайта: service worker управляет только своей
> папкой и тем, что ниже неё. В `/score/` он не заденет основной сайт, а сайт
> не заденет его.

> Если захочешь отдельный поддомен (`score.<домен>`): в Cloudflare это
> настраивается дополнительно, но для старта подпапка проще и работает сразу.

## Проверка, что PWA установилась правильно

На компьютере открой `https://<домен>/score/` в Chrome → **F12** → вкладка
**Application**:
- **Manifest** — показывает имя «Своя Подача», иконки, цвета, без ошибок.
- **Service Workers** — статус *activated and running*.
- В адресной строке появляется значок установки (⊕).

Быстрый аудит: **Lighthouse** → *PWA* → должно быть «Installable».

## Установка на телефон

Условие: сайт открыт по **https://** (Cloudflare даёт HTTPS автоматически).
Просто скачанный HTML-файл так не поставится — нужен именно адрес сайта.

### Android (Chrome)
1. Открой `https://<домен>/score/` в Chrome.
2. Обычно внизу появляется плашка **«Установить приложение»** — нажми.
3. Если нет: меню **⋮** → **«Установить приложение»** / **«Добавить на главный
   экран»**.
4. Иконка «Своя Подача» появится среди приложений, запуск — в полный экран без
   адресной строки.

### iPhone / iPad (только Safari!)
1. Открой адрес **именно в Safari** (в Chrome на iOS установка недоступна —
   ограничение Apple).
2. Кнопка **«Поделиться»** (квадрат со стрелкой вверх).
3. **«На экран “Домой”»** → **«Добавить»**.
4. Иконка появится на домашнем экране, откроется как отдельное приложение.

### Часы (Apple Watch / Wear OS) — честно
Полноценную PWA на часы поставить **нельзя**: ни watchOS, ни Wear OS не
устанавливают веб-приложения как нативные. Варианты:
- держать приложение открытым на телефоне рядом с кортом;
- если счёт на часах реально нужен — это отдельный **нативный** проект
  (SwiftUI для Apple Watch / Kotlin + Wear OS). Скажи — оценю отдельно.

## Обновление приложения

Когда меняешь `index.html` или иконки — открой `service-worker.js` и подними
версию кэша:
```js
const CACHE_VERSION = 'picklepoint-v2';  // было v1
```
Запушь. У пользователей при следующем заходе подтянется новая версия, старый
кэш удалится сам.

## Настройка под себя

- **Название на иконке:** уже стоит «Своя Подача». Меняется в `manifest.json`
  (`name`, `short_name`) и в мета-теге `apple-mobile-web-app-title` в
  `index.html`.
- **Язык по умолчанию:** сейчас русский. В `index.html`, модуль `I18N`, строка
  `let current = 'ru'`. Пользователь может переключить кнопкой **RU/EN** —
  выбор запоминается.
- **Иконка:** замени файлы в `icons/` на свои под теми же именами и размерами.
  Промпты для генерации логотипа — ниже.

## Адаптивность

Приоритет — телефон в портрете; корректно на планшете и десктопе; учтены
«безопасные зоны» (вырезы/челки); нет горизонтального скролла; тач-таргеты
≥ 64px. Клавиатурные шорткаты: **Пробел / ↑** — очко, **↓ / F** — ошибка,
**Ctrl+Z** — отмена, **N** — новая игра, **M** — звук. Поддержан режим
уменьшенной анимации.
